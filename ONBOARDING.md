# Dr. Offline: The Offline Medical Reasoning Engine - Onboarding Guide

Welcome to the Dr. Offline development team! This document serves as an onboarding guide for new contributors to the project. We are participating in the **2026 Gemma 4 Good Hackathon**, aiming to build a fully offline medical RAG system.

---

## 🏆 Project Overview

Our project targets the **Global Resilience** track.
Our core philosophy: **Providing a network-independent "omniscient medical expert" for remote areas, disaster zones, or resource-deprived environments.**
We aim to demonstrate that through Edge AI engineering optimization, Gemma 4, paired with an offline database, can run smoothly on standard edge devices, providing grounded medical advice to frontline medical staff while avoiding AI hallucinations.

---

## 🏗️ Core Tech Stack

We have adopted **Pure Python + Kiwix** as our primary architecture.

* **LLM Engine (The Brain)**: **Gemma 4 26B MoE (4-bit Quantization)**. We utilize MoE architectures with 4-bit quantization to compress the VRAM requirement from ~52GB to 15-18GB. We use **llama.cpp** to run the model.
* **Knowledge Source (Sole Source of Truth)**: **Kiwix (.zim format) + WikiMed**. Kiwix provides highly compressed local `.zim` files, perfectly solving the storage problem of medical encyclopedias in offline environments. We use `kiwix-serve` for low-level full-text retrieval.

---

## 🤖 Agentic Workflow

Our system is an Agent capable of reasoning. Here is an example workflow when a user inputs a medical query:

1. **Analysis & Query Expansion**:
   - User Input: "My chest feels like there's a heavy rock on it."
   - Gemma 4 translates colloquial symptoms into a Query Expansion instead of directly searching.
   - Gemma Output (Search terms): `("chest pain" OR "myocardial infarction" OR "angina pectoris")`.

2. **Kiwix Search**:
   - The Python script calls the `kiwix-serve` API, using the keywords generated by Gemma 4 to accurately locate relevant WikiMed articles.

3. **Retrieval-Augmented Generation**:
   - The full texts of the Top 3 retrieved articles are injected into Gemma 4's 256K Context Window.
   - Gemma 4 cross-references the documents and cites the sources (Grounded output), providing the final, hallucination-free medical advice to the user.
